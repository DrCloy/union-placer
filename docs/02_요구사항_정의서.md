# 요구사항 정의서

## 1. 기능적 요구사항

### 1.1 블록 입력

#### 1.1.1 API 조회

- 넥슨 ID 로그인으로 계정 정보 조회
- 닉네임 입력으로 해당 월드 유니온 정보 조회
- (상세 방식은 API 구조 확인 후 확정)

#### 1.1.2 캐릭터 선택

- 월드 단위로 캐릭터 목록 표시
- 다중 선택 가능 (카드 형태 UI)
- 선택 시 블록 종류별 자동 카운팅

#### 1.1.3 수동 입력

- API 없이 블록 종류별 개수 직접 입력 가능

### 1.2 목표 칸 수 입력

- 숫자 직접 입력
- 선택한 블록 총합 이하로 제한
- 0으로 초기화 버튼
- 최대로 채우기 버튼

### 1.3 우선순위 설정

#### 1.3.1 프리셋

- 사냥: 획득 경험치 > 크리티컬 데미지 > 일반 데미지
- 보스: 크리티컬 데미지 > 보스 데미지 > 방어력 무시

#### 1.3.2 커스텀

- 사용자가 영역별 우선순위 직접 설정
- 0순위: 필수 영역 (무조건 채움)
- 1순위 이하: 중요도 순서

#### 1.3.3 내부 영역

- 외부 영역 우선 채운 후 내부 순차 채우기
- 블록 연결 유지 범위 내에서 사용

### 1.4 자동 배치 탐색

#### 1.4.1 배치 규칙

- 모든 블록은 상하좌우로 연결되어야 함
- 각 블록의 기준점이 중앙 4칸 중 1칸에 위치해야 배치 가능

#### 1.4.2 탐색

- 목표 칸 수 + 우선순위 기반 최적 배치 탐색
- 결과는 1개만 표시
- 로딩 표시

#### 1.4.3 대안 추천

- 불가능 시 우선순위 낮은 영역부터 칸 조절
- 조절한 칸은 더 낮은 순위 또는 내부 영역으로 이동
- 대안도 1개만 표시

### 1.5 결과 시각화

- 유니온 판에 배치 결과 표시
- 블록 구분: 색상 + 테두리

## 2. 비기능적 요구사항

### 2.1 성능

- 탐색 시간 제한 없음
- 사용자가 탐색 중단 가능

### 2.2 호환성

- 모던 브라우저 전체 지원 (IE 제외)
- 모바일 웹 접속 대응

### 2.3 오프라인

- 선택 사항 (우선순위 낮음)
